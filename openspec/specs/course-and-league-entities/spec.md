## ADDED Requirements

### Requirement: Course entity
The system SHALL persist courses with a name and timezone. Course setup is performed via SQL by a super admin.

#### Scenario: Course record exists
- **WHEN** the database is seeded with a course record
- **THEN** the API can read the course by its UUID primary key

### Requirement: TeeBox entity
The system SHALL persist tee boxes belonging to a course.

#### Scenario: Tee box belongs to course
- **WHEN** a tee box record is created
- **THEN** it references exactly one course

### Requirement: Hole entity
The system SHALL persist holes belonging to a course with a hole number, par, and handicap index.

#### Scenario: Hole data is complete
- **WHEN** a hole record exists
- **THEN** it carries a hole number, par value, and handicap index, all referencing exactly one course

### Requirement: League entity
The system SHALL persist leagues belonging to a course with a name, day of week, and default round length.

#### Scenario: League belongs to course
- **WHEN** a league record is created
- **THEN** it references exactly one course

### Requirement: LeagueConfiguration entity
The system SHALL persist one configuration record per league specifying the handicap system and whether subs are allowed.

#### Scenario: Configuration is one-to-one with league
- **WHEN** a league record exists
- **THEN** exactly one LeagueConfiguration record references it

### Requirement: Season entity
The system SHALL persist seasons belonging to a league, each with a year, start date, end date, and a nullable ArchivedAt for soft delete.

#### Scenario: Season belongs to league
- **WHEN** a season record is created
- **THEN** it references exactly one league

#### Scenario: Season soft delete
- **WHEN** a season's ArchivedAt is set to a non-null timestamp
- **THEN** the season is considered archived and excluded from active queries

### Requirement: LeagueMembership entity
The system SHALL persist league memberships joining a golfer to a season, carrying the golfer's handicap for that season, a commissioner flag, and a nullable ArchivedAt.

#### Scenario: Membership links golfer to season
- **WHEN** a league membership record is created
- **THEN** it references exactly one golfer and exactly one season

#### Scenario: Membership soft delete
- **WHEN** a membership's ArchivedAt is set
- **THEN** it is excluded from active membership queries

### Requirement: UUID primary keys
All entities SHALL use UUID primary keys generated by the database.

#### Scenario: PK is generated on insert
- **WHEN** a record is inserted without specifying a primary key
- **THEN** the database assigns a UUID value

### Requirement: Audit timestamps
All entities SHALL carry CreatedAt and UpdatedAt timestamps stored in UTC.

#### Scenario: Timestamps set on insert
- **WHEN** a record is inserted
- **THEN** CreatedAt and UpdatedAt are set to the current UTC time

#### Scenario: UpdatedAt advances on update
- **WHEN** a record is updated
- **THEN** UpdatedAt reflects the time of the update

### Requirement: Soft delete fields
Golfer, Season, LeagueMembership, and Team SHALL carry a nullable ArchivedAt timestamp. Records with a non-null ArchivedAt are excluded from active queries.

#### Scenario: Archived record excluded
- **WHEN** a record's ArchivedAt is set
- **THEN** queries for active records do not return it

### Requirement: SQL seed template
The system SHALL provide a SQL seed template that a super admin can run to create a course, tee boxes, holes, a league, a league configuration, a season, and at least one golfer.

#### Scenario: Seed script is runnable
- **WHEN** the super admin executes the seed SQL against the running database
- **THEN** the foundational records are created and the API can read them
